# Self-Improvement Policy

The agent may propose improvements. It may not directly promote them.

Allowed proposals:

- add an eval case for a discovered failure
- clarify a skill instruction
- tighten a policy
- add a receipt field
- add a runtime adapter check

Promotion requires:

1. Evidence of the failure or missed opportunity.
2. A new or updated eval case.
3. Validator pass.
4. Eval runner pass.
5. Human approval or a future explicit governance policy.

Blocked:

- self-editing installed skills in place
- deleting evals to make a failure disappear
- weakening safety policies to improve convenience
- storing unverified lessons as memory
