# Memory Policy

Memory is allowed only when it is curated.

Every memory entry must include:

- source
- date captured
- confidence
- expiry or refresh condition
- reason it helps future runs
- whether it was verified in the current run

Do not store:

- raw chat dumps
- secrets
- unverifiable claims
- temporary emotions or preferences
- repo state that can drift without an expiry

Memory can guide what to check first. It cannot replace current evidence.
